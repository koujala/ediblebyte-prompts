# Apply Then Find the Edge

Use this immediately after reading something and feeling like you understood it. The prompt tests that feeling by asking AI to pose application problems — including a near-miss case where the concept seems to apply but doesn't — before giving any feedback.

The risk it addresses: the fluency illusion. Reading something clearly feels like understanding it. This prompt forces the difference to become visible before a high-stakes moment does it instead.

---

## The prompt

```
I just read about [CONCEPT]. Here's what I think I understood: [YOUR SUMMARY IN YOUR OWN WORDS — not copied from the source, written fresh].

Don't tell me yet if I'm right. Give me a realistic problem where I have to apply this concept — then give me a second one where it seems like it should apply but actually doesn't.

After I've worked through both, show me where my understanding held and where it was still pattern recognition dressed up as comprehension.
```

---

**How to use:** Write the summary fresh, from memory, without looking back at the source. That act alone is a retrieval test. Work through both problems before reading the debrief — the temptation to skip to the answer is the same avoidance the prompt is designed to catch.

**What this reveals:** Whether comprehension is genuine (survives application to novel problems) or shallow (breaks down when the surface looks different). The second problem — the near-miss — is specifically designed to expose the edges of the concept, which is where real understanding lives.
