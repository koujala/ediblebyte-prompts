# Build Without the Scaffold

Use this immediately after finishing a tutorial — before the next one starts. The prompt asks AI to give you only the end-goal requirements and then act as a Socratic tutor: minimum hints, no solutions, honest comparison at the end.

The risk it addresses: the tutorial plateau. This is the active escape route — rebuilding from a blank file to separate what you actually retained from what the step-by-step format was carrying.

---

## The prompt

```
I just finished a tutorial on [TOPIC — e.g. "building a REST API with FastAPI", "React state management with Redux"]. I want to rebuild the core project from memory without looking at the tutorial.

Give me only the end-goal requirements — nothing else. As I work through it, if I get stuck and ask for a hint, give me the smallest possible nudge rather than the solution.

When I am done, compare what I built to what a solid implementation would look like and tell me specifically what I missed.
```

---

**How to use:** Close the tutorial before you start. The exercise only has value if you're genuinely working without the scaffold. When you ask for hints, be specific about where you're stuck — this forces the smallest useful nudge rather than the answer. The final comparison is the most important part: it names the gaps rather than just flagging that they exist.

**What this reveals:** What you retained from the tutorial vs. what the tutorial was providing through its structure. Projects rebuilt from memory are almost always smaller and simpler than the tutorial version — and the delta between the two is exactly the learning that hasn't happened yet.
