# Recreate the Friction

Use this when you've encountered a concept only through summaries, overviews, or key-point extracts — and you want to build actual understanding rather than label-familiarity. The prompt asks AI to make the concept hard, not to explain it.

The risk it addresses: the summary shortcut. Consuming conclusions instead of struggling with the source produces ideas that exist as vocabulary, not as working knowledge. This prompt reconstructs the friction the summary bypassed.

---

## The prompt

```
I need to understand [CONCEPT] properly, not just be able to talk about it. Don't explain it to me.

Give me a realistic scenario where I have to work it out from first principles — withhold the answer until I've reasoned through it, even if I get it wrong. Then give me a second scenario where the same concept applies but the surface looks different enough that I can't just pattern-match from the first one.

I want to know when I've actually understood it, not just when I've seen it once.
```

---

**How to use:** Resist the pull to ask for an explanation before attempting the scenarios. The prompt only works if you commit to the reasoning before seeing the answer. If you get it wrong — good. That's the friction producing the learning.

**What this reveals:** Whether the concept is genuinely understood or whether you've been working with a label that runs out when the situation gets novel. The second scenario specifically tests transfer — the thing summaries are worst at building.
