# Phase 3 — Claude Code: Build & Ship (Days 22–30)

Prompts for exploring the environment, scoping a project, building it, deploying it, and reflecting on the month.

---

## Day 22 — Explore before you build

**Understand the environment:**
```
Create a simple 'hello world' HTML file in this project folder. Then explain to me what you just did, where the file is, and how I'd open it in a browser.
```

**Read existing code:**
```
Look at the files in this project. Give me a plain-language overview of what this project does, how it's structured, and what the main files are responsible for.
```

**Make a small change:**
```
Find [a specific piece of text / a button / a heading] in the code and change it to [something new]. Show me what you changed and explain where to find it.
```

---

## Day 23 — Scope your project

**Scope check:**
```
I want to build [idea]. I have [no coding experience / some basic knowledge / comfortable with code but new to this area]. Is this scope realistic for one week? If not, what's a smaller version that still delivers value? Give me a stripped-down spec — what it does, what it doesn't do, and what I'd need to get it running.
```

**Plan the architecture:**
```
Before we start building, help me think through the approach for [project]. What are the main components? What tech stack would you recommend for someone at my level? What do we build first and why?
```

---

## Days 24–26 — Build it

**Day 24 — start:**
```
Let's start building [project]. Begin with [first component from the plan]. Write the code, explain what each part does in plain language, and tell me what I need to do to run it locally.
```

**End of each day:**
```
Here's where we are: [paste what's working]. What's the next thing to build? What could break? What should I test before we continue?
```

**When something breaks:**
```
This is the error I'm getting: [paste error]. Here's the relevant code: [paste code]. What's wrong and how do I fix it? Explain it so I understand what happened, not just how to fix it.
```

**When deciding between options:**
```
I'm deciding between [option A] and [option B] for [part of the project]. What are the trade-offs? What would you recommend for someone at my level building this specific thing?
```

---

## Days 27–29 — Deploy and polish

**Day 27 — deploy:**
```
I want to deploy [project] to [platform — AWS Amplify / Netlify / Vercel / GitHub Pages]. I've never deployed before. Walk me through the steps from where I am now to a live URL. Flag anything that could go wrong.
```

**Day 28 — if issues:**
```
The deployment failed with [error]. Here's the build log: [paste relevant output]. What's causing this and how do I fix it?
```

**Day 29 — polish:**
```
Here's what I've built: [brief description]. What are the 3 most important things to improve before I share it? Focus on things that affect whether someone else can understand and use it — not perfection.
```

---

## Day 30 — Retrospective

```
I've just completed 30 days of intensive work with Claude across three surfaces. Here's what I built and learned: [summarise]. Ask me questions to help me reflect on: (1) what I actually know now vs. Day 1, (2) what changed in how I work, (3) what I'd do differently, (4) what I want to go deeper on next.
```
