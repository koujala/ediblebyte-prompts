# Report Skeleton

Most people write reports linearly — starting at the beginning and discovering the structure as they go. The result usually buries the most important point somewhere in the middle. This prompt gets you the structure *before* you write a word, so you write into a shape that leads with impact.

```
I need to write a [REPORT / SUMMARY / PROPOSAL / MEMO] on [TOPIC] for [AUDIENCE].

The key point I need to land: [THE SINGLE MOST IMPORTANT THING]
Secondary points: [Y AND Z]
The reader is: [SCEPTICAL / SUPPORTIVE / UNFAMILIAR / IMPATIENT / SENIOR]
What I need them to do after reading: [DECISION / APPROVAL / NOTHING — JUST KNOW / NEXT MEETING]

Suggest a structure that:
- leads with impact (the key point, or the reader's problem — not my context)
- builds the case efficiently without padding
- addresses the single objection this reader is most likely to raise
- ends with a clear recommendation or next step

Do not write the content yet. Just give me:
1. A section-by-section skeleton with a one-line description of what each section does.
2. For each section, the one thing that absolutely has to appear in it, even if everything else gets cut.
3. Which section will be the hardest to write — and why.
```

**How to use the output:** Once you have a skeleton that works, writing into it is dramatically faster than writing from scratch. Treat the skeleton as the real thinking. The prose is just filling in structure you already know is sound.
