# Wall 5 Fix — Diagnose Before You Give Up: A Bad Output Means Try Again Differently

From Post 5 of the Your AI Path series: the corrective move for stopping when one thing doesn't work.

**The stall:** You ask the model to do something. The output is wrong, generic, or confidently incorrect. You feel the tool is overhyped. You close the tab and don't come back for a week.

**The problem:** This is the most common and most costly stall. Almost every bad output is a prompting issue, not a model limitation. Language models fail in predictable ways — and predictable failures are fixable.

## The predictable failures and their fixes

**Hallucinating specific facts:**
```
Work only from the information I've provided. If you're uncertain about any fact, flag it explicitly rather than stating it as true.
```

**Generic output from a generic prompt:**
```
This is too generic. I need this to be specific to [your specific situation — add the context you left out the first time].
```

**Wrong tone:**
```
The tone is wrong. I need this to sound [specific description — e.g., more direct, less formal, warmer, more confident]. Rewrite with that tone only.
```

**Too much text, not enough specificity:**
```
This is too long and too broad. Cut it to [target length] and focus only on [the specific thing that matters].
```

**The simplest fix that works most of the time:**
```
That's not quite right. Here's what's wrong with it: [explain specifically what's off]. Try again.
```

## The rule

A bad output means try again differently — not stop entirely. Prompting is a skill. It gets better with use. Most bad outputs become good outputs two messages later.

The model doesn't get offended. It just tries again.
