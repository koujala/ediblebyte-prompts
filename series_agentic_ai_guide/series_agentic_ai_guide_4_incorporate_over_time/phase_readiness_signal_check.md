# Phase Readiness Signal Check

Evaluate whether your production data justifies moving from the current phase to the next. Each capability in the phase map has a specific trigger the article identifies. Building before the trigger fires means building blind.

From the article: "Each phase builds on evidence generated by the previous phase. You cannot correctly configure a RAG pipeline without knowing which knowledge gaps are causing escalations. You cannot safely expand autonomy without months of data on what the agent gets right."

---

## Phase 2 Readiness — Months 3–6

Run this check after 4–6 weeks of production operation.

```
My enterprise AI agent has been running in production for [NUMBER] weeks. I need to assess whether my production data justifies adding Phase 2 capabilities: Memory, RAG, and an Evaluation Framework.

**Current baseline metrics (from production):**
- Daily request volume: [NUMBER]
- Task completion rate (autonomous, no escalation): [%]
- Escalation rate: [%]
- Average latency: [MS]
- Daily cost: [AMOUNT]

---

**Memory — Persistent Context Across Sessions**

The signal to watch for: users re-explaining context they've already provided in prior sessions — prefacing requests with "like last time" or "same issue as before."

My data:
- Have users complained about re-explaining context? [YES / NO / UNCLEAR]
- What % of tickets/interactions reference a previous session? [NUMBER OR ESTIMATE]
- Do I have a vector database or memory store provisioned yet? [YES / NO]

Assess: Does the evidence justify building cross-session memory now? What is the signal strength — clear, emerging, or absent?

---

**RAG — Grounding the Agent in Your Data**

The signal to watch for: the agent's escalation rate for knowledge-intensive queries is significantly higher than for structured tool-call queries. Escalation reasons cluster around "I don't have that information."

My data:
- What are the top 3 reasons cases escalate to a human? [LIST]
- What % of escalations are due to knowledge gaps vs. policy/complexity? [ESTIMATE]
- Which specific document types or knowledge bases are users asking about that the agent cannot address? [LIST]

Assess: Does the evidence justify building RAG now? What specific knowledge gaps would the retrieval layer address — based on actual escalation patterns, not anticipated future needs?

---

**Evaluation Framework — Systematic Quality Measurement**

The signal: you have 4–6 weeks of production data and a baseline, but no systematic way to know whether changes you make are improvements or regressions.

My data:
- Do I have a baseline task completion rate from week 1? [YES / NO]
- Can I currently tell whether a prompt change improved or degraded performance? [YES / NO / ONLY BY FEEL]
- Am I running any human-validated accuracy sampling on agent outputs? [YES — HOW OFTEN / NO]

Assess: Is an evaluation framework justified now? What specific metrics should the framework track given this use case?

---

**Overall Phase 2 recommendation:**
Based on the signals above, which Phase 2 capabilities are justified now, which should wait, and what additional evidence would strengthen the case for the deferred ones?
```

## Phase 3 Readiness — Months 6–12

Run this check after 6+ months of production operation and before committing to orchestration or state management.

```
My enterprise AI agent has been in production for [NUMBER] months. I need to assess whether we are ready for Phase 3 capabilities: Multi-Agent Orchestration, Durable State Management, and Autonomy Expansion.

**Current production state:**
- Months in production: [NUMBER]
- Average daily volume: [NUMBER]
- Task completion rate (autonomous): [%]
- Escalation rate: [%]
- Human review decisions logged to date: [NUMBER]

---

**Multi-Agent Orchestration**

The trigger: the single agent is hitting the limits of what one context window can handle, OR the task genuinely requires parallel research across multiple domains.

My situation:
- Are there tasks where the agent visibly degrades due to context window limitations? [DESCRIBE OR "NO"]
- Are there tasks that require parallel work across domains that one agent cannot handle sequentially? [DESCRIBE OR "NO"]
- Current token usage per complex interaction: [ESTIMATE]

Note from the article: multi-agent systems consume up to 15x more tokens on average and introduce coordination failure modes (circular loops, conflicting decisions, silent handoff failures) that don't exist in single-agent systems.

Assess: Has orchestration earned its place based on these signals, or is the complexity cost still higher than the value?

---

**Durable State Management**

The trigger: workflows are extending beyond a single session — tasks that span hours, days, or multiple approval steps.

My situation:
- Are there workflows that currently fail or lose state when the session ends? [DESCRIBE OR "NO"]
- What is the longest-running workflow currently in production? [DURATION]
- Are users or downstream systems affected by state loss across sessions? [YES / NO]

Assess: Is durable state management justified, or is the context window sufficient for current workflow lengths?

---

**Autonomy Expansion**

The trigger: 6+ months of production data showing consistent accuracy for specific action types, with a clear owner for the governance decision.

My situation:
- Which specific action types is the agent currently routing to humans? [LIST]
- For each action type: what is the agent's accuracy rate when humans have reviewed its proposed decisions? [DATA]
- Which action types have enough data (volume + accuracy) to make an evidence-based autonomy case? [LIST]
- Who is the business owner who will sign off on expanding autonomy? [NAME / ROLE]

Assess: Which action types, if any, are ready for autonomy expansion — and what is the proposed monitoring window after each change?
```

---

**How to use:** Run the Phase 2 check at your 4–6 week production review. Run the Phase 3 check at your 6-month review. The output is a go/no-go recommendation per capability — not a binary "move to the next phase" decision. Capabilities within a phase can be added independently when their specific signals appear.
