# Pre-Code Checklist

Run this checklist before your first sprint begins. If any item cannot be checked, that item is your first priority — not the code.

From the article: "If you can check every box, you're ready to build. If you can't, the missing item is your first priority — not the code."

---

## Checklist

```
Pre-Sprint Readiness Check — Enterprise Agentic AI MVP

Agent / Project: [NAME OF AGENT AND USE CASE]
Date: [YYYY-MM-DD]
Reviewed by: [NAMES]

[ ] 1. WORKFLOW SCOPE DOCUMENTED
    The agent's workflow scope is documented in plain language.
    A non-technical stakeholder can read it and understand exactly
    what the agent does and does not do.
    Document location: [LINK OR FILE PATH]

[ ] 2. SUCCESS CRITERIA DEFINED
    Success criteria are defined with a specific metric and a
    90-day target — not "users find it helpful."
    Metric: [E.G., "Reduce tier-1 escalations by 30%"]
    Target date for first measurement: [YYYY-MM-DD]

[ ] 3. "NEVER WITHOUT HUMAN REVIEW" LIST EXISTS
    A written list of actions the agent must never take without
    human approval has been created and reviewed by at least
    one business stakeholder.
    Reviewed by: [NAME AND ROLE]
    Document location: [LINK OR FILE PATH]

[ ] 4. USE CASE IS NARROW, HIGH-IMPACT, AND NON-CATASTROPHIC
    The use case has high volume, clear success/failure criteria,
    and failure modes that are recoverable — not front-page news.
    Fitness check completed: [ ] Yes  [ ] No (complete it first)

[ ] 5. HUMAN VALIDATOR IDENTIFIED
    At least one human currently does this work and is available
    to validate agent outputs during the MVP phase.
    Name and role: [NAME — ROLE]
    Availability commitment: [E.G., "2 hours/week for 90 days"]

[ ] 6. SYSTEM ACCESS PATH EXISTS
    You know which systems the agent will need access to,
    and you have a credible path to obtaining that access
    before the agent needs it.
    Systems required: [LIST]
    Access status: [E.G., "IT ticket submitted," "Approved," "TBD"]
```

---

**How to use:** Complete this as a team, in writing, before sprint planning. Store it alongside your agent job definition document. When a stakeholder asks "how did you decide to build this?" — this document is part of your answer.

**What this is not:** A gate that requires perfect answers before any work begins. It is a forcing function for having the right conversations early, while changing direction is cheap.
